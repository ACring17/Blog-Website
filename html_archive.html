Coding Fundamentals: Consistency is Key

Feb. 17, 2021

With coding fundamentals I will be talking about key lessons learned while in my coding journey. Ideas and topics that will be helpful to revisit and teach other learn lessons that are important to me. The first one will be about the lesson of perseverance and consistency. These lessons and skills are not only important for coding but in life in general.

I have found that coding is very similar to fitness, in that to become great and be the envy of everyone in the field is a long journey of working hard and being consistent. To see results with weight training one must be consistent with lifting heavy weights on a regular baisis. To be able to run a marathon one must have regular running routine. similarly I have learned that in order to learn code one must find a routine and practice regularly. I have found that when I am consistent with my coding practice the skills seem to stick in my head and become natural to me. I create a new route for my brin to function.

When I take a break or lose motivation from coding I find it to be hard to get back into the rythm of learning and practicing. Through just getting a little bit done for coding I can use the 1% of getting better and let it grow over time. That 1% will continue to grow each day and compound into mastery of the language. The goal should not be to rush through the lessons and learn the concepts fast, but to truly learn the lesson and take my time learning the concepts. When I feel lack of motivation I need to remember that action will help show me success, which will in turn help me create motivation and momentum in learning. Even if you feel dumb, keep learning and practicing. Eventually it will amount into mastery!


Project Idea #1: Change of address directory

Feb. 10, 2021

I have been so busy working on fine-tuning this website that I have been forgetting to log enrties in along the way. This post will be the the first article for the project ideas page. This project came to me as I am getting ready to move this weekend. The idea is it will be a directory of all the organizations and services needed to be contacted for a move. The idea came to me as I stressed over thinking about all my mail, voter registration, and setting up the utilities. I have not done any research into it but it seems like it would be a handy tool to have. All the services and contact information for everything needed for a move.

The idea is to create a site that is an all in one site where the user logs-in and can track all the different authorities they need to contact about a change of address. List the agencies name, the current address they have listed and a link to where the user must go to change the address listed. For mail, voting, deliveries, MVD, car registration and such like that. A quick way for those who move to keep all their information up to date. Creating submition forms with all the relevant information and then creating forms for the user to see all the services and organizations needed to contact before the list. Almost like a todo list for moving, but the site comes with all the relevant contact information.

It seems like a fun twist on a normal todo list while also incorporating fun new lessons and challenges for me to take on. This idea has beeen fun to think and flush out and cannot wait to get started on it.

Learning CSS: The Basics

Jan. 13, 2021

Cascading Style Sheets(CSS) has been a little more challenging than HTML to learn. It has to do with the depth at which CSS can alter the HTML document. CSS is where the styling and art of computer programming comes together to alter the appearance of the website for the user. If HTML is the bones, than CSS is the skin and appearance of the body. CSS allows for a set of rules to be implemented for different element tags. I am learning that CSS has a lot of depth and can be implemented in a lot of creative ways to really make a web site dynamic and visualy appealing to the user. CSS can also make the website easier to navigate and be user friendly for people with disabilities.

CSS is another mark-up language and is seems to be pretty straightforward. List the element you would like to target and then list the changes you would like the browser to apply. Where it gets tricky is knowing the scope of the which rules. Some rules can be set globaly to apply to the whole body tag, while also having the ability to set up IDs to specificly target a certain tag within the HTML. It comes down to what the programer wants to edit and adjust on the website.

I feel like there is so much more to CSS I can still learn. That I am only scratching the surface of it's potential. So far CSS has been my favorite language to learn and implement.
